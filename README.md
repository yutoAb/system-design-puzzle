# システム設計パズル

システム設計面接に出てくるようなアーキテクチャを、パズルのピースを組み合わせて構築するWebアプリゲーム。

## コンセプト

- お題が与えられる（例:「100万人が同時視聴する動画配信サービスを設計せよ」）
- パズルのピース（CDN、ロードバランサー、DB、キャッシュ、メッセージキューなど）を組み合わせる
- 正解は1つではなく、コスト・可用性・レイテンシのトレードオフがある

## 題材の例

- 動画配信サービス
- チャットサービス
- SNS
- MLOpsパイプライン
- PB級データシステム

## 動機

- システム設計面接の勉強
- エンジニア能力向上
- 個人開発・マネタイズ

## 開発

```bash
npm install
npm test          # ドメイン層のテスト (Node 標準 test runner)
npm run dev       # API + Vite 開発サーバを並行起動 → http://localhost:5173
npm run preview   # 本番ビルドをローカル確認 → http://localhost:4173
```

## 遊び方

1. お題の機能要件・非機能要件・面接メモを読む
2. 右側の設計ピースを、左側のボードへドラッグして配置する
3. エッジ / 流量制御 / サービス / 保存 / 非同期 / データ分散の各レーンを埋める
4. アーキテクチャ予算を超えすぎないように調整する
5. `採点する` を押して、配置済みピースのスコア・強み・リスク・面接フィードバックを見る

## アーキテクチャ

クリーンアーキテクチャを採用し、依存方向を内側に向けます。

- `src/domain`: 採点ルール、チャレンジ、コンポーネント定義
- `src/application`: ユースケース
- `src/infrastructure`: インメモリリポジトリ
- `src/interface-adapters`: コントローラ
- `client`: React + Vite + React Flow のブラウザ UI
- `api`: Vercel Functions エントリポイント
